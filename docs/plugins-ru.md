# Плагины nameless

Этот документ описывает текущий формат плагинов nameless.

## Что поддерживается сейчас

- `*.plugin` - классический формат.
- `*.js`, `*.mjs`, `*.cjs` - поддержка метаданных из JS-файлов.
- Автоопределение метаданных по тегам `@name`, `@version`, `@description`, `@author`, `@icon`, `@id`, `@permissions`.
- Подключение плагина через список плагинов и через установку из файла.

## Что работает частично

- Выполнение JS-плагинов `*` - метаданные читаются, но полный JS runtime еще не завершен.
- Полная совместимость со сторонними скриптами `*` - зависит от конкретного API, которое они используют.
- Локальные уведомления `*` - в документации плагинов описываются, но как отдельная нативная подсистема еще не добавлены.

## Формат метаданных

Пример для JS-плагина:

```js
// @name nameless Notifications
// @version 1.1
// @description Уведомления, которые запускаются по правилам плагина.
// @author nameless
// @id nameless-notifications
// @permissions notifications, storage, chats
```

## Что можно делать плагинами

- Добавлять настройки в раздел nameless.
- Читать и сохранять состояние плагина.
- Использовать уведомления, если это реализовано в нативной части клиента.
- Работать с UI-элементами и настройками, когда соответствующий хук уже подключен.

## Ограничения

- Плагин не должен считаться полным заменителем системных push-уведомлений.
- Функции, помеченные `*`, требуют дополнительной интеграции и могут быть доступны только частично.
